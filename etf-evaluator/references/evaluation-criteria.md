# ETF 评估维度

## 0. 报告参数说明

### 核心公式

- **超额溢价** = 显示溢价 + 当天涨幅 - 平均涨幅 - 历史平均溢价
  - 超额溢价越低 → 越值得买入（被低估）
  - 超额溢价 > 0 → 溢价偏高（被高估）

- **综合超额** = 1M×35% + 3M×25% + 6M×20% + 1Y×10% + ALL×10%
  - 多时间窗口加权平均，近期权重更高
  - 负值越大 = 越被低估，正面信号

- **分值** = 1Y净值涨幅×10% + (-综合超额)×75% + (-估算溢价)×15%
  - 分值越高 = 越值得买入
  - 净值涨幅高（基金能力强）+ 综合超额低（被低估）+ 当前溢价低（买入成本低）

### 表格字段

| 字段 | 含义 |
|------|------|
| 价格 | ETF 二级市场实时交易价格 |
| 涨幅 | 当日价格涨跌幅 |
| 估算溢价 | 当前价格相对净值的溢价率（含期货修正）|
| 3M/6M/1Y超额(均值) | 该时间窗口的超额溢价(对应时间窗口的历史平均溢价) |
| 综合超额 | 多窗口加权后的综合超额溢价 |
| 1Y净值 | 过去一年的净值涨幅，反映基金跟踪能力 |
| >7%天 | 过去一年中溢价超过7%的交易日天数（正面：说明市场追捧度高） |
| 分值 | 综合评分，越高越推荐 |

### 基准涨幅

- 同指数所有 ETF 当日涨幅的均值
- 纳指和标普分别计算
- 用于计算超额溢价中的"偏离基准"部分

### 期货修正

- 净值 T+1 更新：每日公布的是前一个交易日的净值
- 周末/假日无交易 → 净值不更新，属正常机制
- 期货修正 = 用 NQ/ES 期货涨跌幅弥补未交易日的市场变化
- 累计修正 = 连乘各日期货变化率

## 1. 期货趋势分析

- 读取 NQ/ES 连续 2-3 天的涨跌幅
- 判断短期走势：连续上涨、连续下跌、震荡（先跌后涨/先涨后跌）
- 语言示例："NQ 连续两日先跌后涨（-0.83%→+0.62%），震荡修复中"

## 2. 溢价收敛/发散

- 对比 3M/6M/1Y 超额均值的变化方向
- 6M 超额 < 3M 超额 → 溢价收敛（积极信号）
- 6M 超额 > 3M 超额 → 溢价发散（消极信号）
- 关注综合超额绝对值：< -1.5% 为显著低估，> 0 为溢价偏高

## 3. 推荐变化追踪（需要前日报告）

- 今日推荐 ETF 是否与昨日一致
- 若发生变化：解释排名变化的原因（分值差距、溢价波动）
- 若未变化：强调持续推荐的稳定性

## 4. 风险与机会指标

- **>7% 天数多**（正面信号）：历史高溢价天数多 = 该 ETF 常被市场追捧。当前溢价远低于历史高位时，说明买入窗口大，博取基金溢价回归的收益空间更大
- **估算溢价过高**（> 3.5%）：短期溢价风险，追高成本大

## 5. 输出格式

保存到独立文件 `analysis_{date}_llm.md`：

```markdown
# ETF溢价分析评估 {date}

**市场**: [1-2 句，NQ/ES 走势概括]

**纳指**: [2-3 句，首选 ETF + 理由，次选 ETF + 特点]

**标普**: [2-3 句，首选 ETF + 理由，特殊情况说明]

**风险**: [既说明机会也说明风险，平衡表达]
```

**风险描述示例**：
- "159612 溢价 >7% 天数达 96 天，当前溢价合理，可以博高溢价收益，但也可能有风险，当前溢价最高。"
- "513390 历史仅 13 个月，长周期指标可靠性偏低，但短期数据表现良好。"

## 6. 输出约束

- 总字数 ≤ 300 字
- 交易员早间简报语气：简洁、客观、数据驱动
- 不做涨跌方向预测
- 引用具体数值时直接使用报告中的数据
- 不重复报告中已有的公式说明
- **只使用普通人能理解的指标**：净值涨幅、当前溢价、3M/6M/1Y平均溢价、涨幅、分值
- **禁止使用专业术语**：不出现"超额溢价"、"溢价收敛/发散"、"综合超额"等普通读者无法理解的概念
- 用"当前溢价 vs 历史平均溢价"的对比来表达高低，而非超额溢价
